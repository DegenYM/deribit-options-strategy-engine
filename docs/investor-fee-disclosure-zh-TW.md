# 投資人分潤與計費說明書

**Deribit 期權策略組合**｜版本 1.1｜績效費 10%｜管理費年化 1.0%

> 正式權利義務以雙方簽署之投資管理協議為準。

---

## 一、費用結構總覽

| 項目 | 費率／原則 | 說明 |
|------|------------|------|
| 管理費 | 年化 1.0%（按季結算） | 按 AUM_mgmt 季度平均規模計提 |
| 績效費 | 10% | 僅就超過 HWM 之新創利潤抽取 |
| 計費單元 | 投資人合併 | 所有**策略**授權子帳合併計算 NAV |
| 高水位 HWM | 必備 | **僅綁定 NAV_perf** |
| 費用收取 | 獨立 **Fee 專戶** | 投資人劃轉應付金額；管理方透過該專戶 API（Wallet 讀寫）自行轉帳 |

---

## 二、兩套計費基礎

### NAV_perf（績效 NAV）— 不含備兌現貨

- **定義**：各**策略**子帳 USDC 等價總權益 − 約定備兌現貨市值  
- **用途**：期間損益、HWM、10% 績效費  
- **不計入**：投資人自有 BTC/ETH 備兌現貨及其漲跌  
- **不含** Fee 專戶餘額（費用專戶不參與 NAV_perf／HWM）

### AUM_mgmt（管理費資產）— 含備兌現貨

- **定義**：NAV_perf + 備兌現貨市值  
- **用途**：1% 管理費（現貨佔保證金與託管資源）  

---

## 三、多幣種換算

結算日 23:59 UTC，各**策略**子帳 book 權益 × Deribit 指數價 → USDC 等價。

```
NAV_perf = Σ 策略子帳 USDC 等價權益 − 備兌現貨 USDC 等價
AUM_mgmt = NAV_perf + 備兌現貨 USDC 等價
```

---

## 四、績效費與 HWM

```
可分配利潤 = max(0, NAV_perf,期末 − HWM − 淨申赎調整)
績效費 = 可分配利潤 × 10%
HWM 僅追蹤 NAV_perf，不含現貨
```

---

## 五、費用收取：獨立 Fee 專戶

管理費與績效費**不**自策略子帳內直接扣款，改採下列方式：

### 5.1 Fee 專戶是什麼？

- 投資人在 Deribit 主帳下另開一個**獨立子帳戶**（建議名稱：`fee`），僅用於存放已結算應付費用。  
- **策略程式只連策略子帳**（如 `naked`、`covered_call`）；**不會**操作 Fee 專戶下單。  
- 季結算後，管理方出具 USDC 等價帳單；投資人於期限內自**主帳或策略子帳**將應付金額 **Internal transfer 劃轉至 Fee 專戶**。

### 5.2 投資人要做的事

1. 建立 Fee 專戶子帳（與策略子帳分開）。  
2. 收到帳單後，依金額劃轉 **USDC** 至 Fee 專戶（若帳單允許且雙方同意，亦可先劃 BTC/ETH 再由管理方依結算日指數換算，細節以協議為準）。  
3. 為 Fee 專戶建立**專用 API Key**（見 5.3），透過安全管道交給管理方。  
4. **策略子帳**的 API Key **勿**開啟 Wallet／提款權限。

### 5.3 Fee 專戶 API Key 權限（與策略子帳不同）

| Deribit scope | Fee 專戶 | 策略子帳 |
|---------------|----------|----------|
| account:read | ✅ 要開 | ✅ 要開 |
| trade:read_write | ❌ 不開 | ✅ 要開 |
| wallet:read | （通常隨 read_write 一併） | ⬜ 通常不必 |
| wallet:read_write | ✅ **要開** | ❌ **不要開** |

管理方在確認 Fee 專戶到帳後，可透過 API 或 Deribit 後台自 Fee 專戶**內部轉帳**至管理方指定帳戶；**不**使用投資人策略子帳的 API。

### 5.4 幣別與逾期

- 帳單以結算日 **USDC 等價** 開立；建議投資人直接劃轉 USDC，避免換幣爭議。  
- 逾期未劃轉至 Fee 專戶，管理方得依投資管理協議暫停新開倉，直至結清。

## 六、結算週期（摘要）

| 項目 | 建議約定 |
|------|----------|
| 定期結算 | 每季末（UTC） |
| 帳單出具 | 結算日後約 10 個工作天內 |
| 投資人劃轉 | 帳單出具後約 5 個工作天內轉入 Fee 專戶 |
| 贖回 | 贖回生效日另結算應計費用 |

## 附錄：HWM 是否包含現貨？

**不包含。** HWM 只跟 NAV_perf；現貨跌不直接拉低高水位。管理費無 HWM，按含現貨的 AUM_mgmt 計提。
